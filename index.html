The Problem (Technical Showcase)

Proyek ini adalah simulasi pengujian skala industri (Technical Showcase) yang berfokus pada kualitas sistem web dan layanan API. Skenario mencakup alur kompleks seperti End-to-End E-commerce User Journey (mulai dari pemilihan produk hingga *checkout*) dan API Data Validation (pengujian skema, respons sukses, serta skenario negatif pada sistem manajemen data pengguna).

The Approach

Komponen Teknologi Alasan & Pendekatan
Framework Cucumber + Gherkin Saya menggunakan pendekatan BDD untuk menjembatani komunikasi antara kebutuhan bisnis dan implementasi teknis. Dengan Gherkin, test cases menjadi dokumen hidup yang bisa dibaca non-programmer.
Design Pattern Page Object Model Framework ini dibangun menggunakan Page Object Model (POM) untuk meningkatkan reusability dan maintainability. Ketika ada perubahan UI, saya hanya perlu mengubah satu file objek halaman tanpa merusak skenario pengujian.
Web Automation Selenium WebDriver Memastikan kompatibilitas pengujian *cross-browser* dan memiliki stabilitas yang sangat baik dengan *explicit waits* untuk menangani elemen web dinamis.
API Automation REST-assured Memungkinkan validasi data API secara end-to-end, memastikan integritas kontrak data dan melakukan chaining requests untuk skenario hybrid.
CI/CD Pipeline GitHub Actions Menjalankan regression test secara otomatis setiap kali ada code commit atau Pull Request, memastikan tidak ada fitur yang rusak sebelum masuk ke *branch* utama.
